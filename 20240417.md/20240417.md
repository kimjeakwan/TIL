git
============= 
1. 깃 은 2005년에 리누스 토르발스에 의해 개발된 분산 버전관리 시스템으로 컴퓨터 파일의 변경사화을 추척하고 사용자를 간에 파일에 대한 작업을 조율하는데 사용된다.
2. 주로 여러명의 개발자가 하나의 소프트웨어 개발 프로젝트에 참여할 떄,소스 코드를 관리하는데 사용된다.
---------------------------------------
git 의 장점 
=============
1. 손 쉽게 코드를 주고 받을 수 있으며, 여러 명이 동시에 한 코드를 가지고 작업하는 병렬적인 작업의 진행이 가능하다
2. 분산 버전 관리 이기 떄문에 작업 도중에 이상이 생겨 저장소가 날라가버린다 해도 쉽게 복구할 수 있다.
---------------------------------------
github
==========
 * 깃허브는 분산 버전 관리 툴인 깃를 사용하는 프로젝트를 지원하는 웹호스팅(외부의 서버를 빌려서 기능을 사용하는 것) 서비스이다.
 ---------------------------------------
github 와 git 의 차이점
===============
* git 로컬 저장소를 사용하기 때문에  실시간으로 작업을 공유할 수 없습니다.
* github는 웹 호스팅 서비스 이기 떄문에
실시간으로 공동 작업 을 할수 있다.
--------------------------------
git bash를 사용해 최종적으로 git push 하는 법
============================================
1. git bash를 통해 해당 파일로 이동을한다.
```
cd "Desktop"
```
```
cd "파일명"
```
하면은 해당 파일로 이동해진다.
```
mkdir "파일명"
```
> 디렉토리를 생산한다는 뜻이다.
> 그후 앞서 설명한 것처럼 파일을 이동한다. 
--------------------------------------  
2. git clone
 - GitHub repository에 있는 내용을 내 로컬(컴퓨터)에 복제하는 명령어다.
 ```
 git clone "github에 Repository주소"
 ```
 3. vscode 를 실행시켜 open Folder눌러 현재 작업중인 파일를 선택한다.
 4. vscode에서 git bash를 실행하고 그 파일로 이동한다.
 5. git add 
 -  staging area 에 변경분을 모아놓는 작업이다.
 - staing area은   스테이징 영역(Staging area)란 commit이 가능한 영역으로, 커밋하기 전 파일을 담아두는 상자라고 볼 수 있다

 ```
 git add "파일명"
 ```
 6. git commit 
 - git commit은 saging area에 저장되어있는 변경 사항들을 로컬저장소에 등록하는 것이다.
 ```
 git commit -m "커밋메세지"
 ```
 7. git push
 - commit 을 통해 staing area을 로컬저장소 등록을 했다.
 그리고 git push 를 통해 로컬저장소 에서  원격 저장소에 업로드 하는 것 이다.
